# <a name="insights"></a>Approfondimenti

SharePoint Online è una raccolta di tecnologie e strumenti basati sul Web che consentono all'organizzazione di archiviare, condividere e gestire le informazioni digitali. Basato su Microsoft SharePoint Server 2013, questo servizio ospitato è ideale per lavorare a progetti, archiviare dati e documenti in una posizione centrale e condividere informazioni con altri utenti. Le funzionalità Insights seguenti consentono alle organizzazioni di utilizzare le informazioni nei database, rapporti e applicazioni aziendali per soddisfare le specifiche esigenze delle organizzazioni.
  
## <a name="business-intelligence-center"></a>Centro business intelligence

Il sito Centro Business Intelligence consente di archiviare connessioni dati, cartelle di lavoro, report, scorecard e dashboard in una posizione centrale e sicura. Ulteriori informazioni sulla [configurazione di un Centro business intelligence](https://go.microsoft.com/fwlink/p/?LinkId=270883).
  
## <a name="calculated-measures-and-members"></a>Misure e membri calcolati

Quando si creano rapporti tabella e grafico pivot che utilizzano i dati archiviati in SQL Server Analysis Services, è possibile creare misure e membri calcolati per i rapporti. Ulteriori informazioni su [Funzionalità di business intelligence in Excel Services](https://go.microsoft.com/fwlink/p/?LinkId=271025).
  
## <a name="data-connection-library"></a>Raccolta connessioni dati

Una raccolta connessioni dati è una raccolta di SharePoint speciale progettata per archiviare e gestire le connessioni dati condivise. Possono includere file ODC creati utilizzando Excel. Per ulteriori informazioni, vedere [raccolte di connessioni dati attendibili in SharePoint Server 2013](https://go.microsoft.com/fwlink/p/?LinkId=271026).
  
## <a name="decoupled-pivottables-and-pivotcharts"></a>Tabelle e grafici pivot disaccoppiati

Quando si crea un rapporto su un grafico pivot, viene creata automaticamente una tabella pivot corrispondente. In Excel 2013, non è necessario visualizzare un rapporto di tabella pivot nello stesso foglio di lavoro del rapporto di grafico pivot corrispondente. È possibile scegliere di visualizzare solo il rapporto del grafico pivot. Questa funzionalità è denominata *tabelle pivot e grafici pivot disaccoppiate*. Ulteriori informazioni su [Tabelle e grafici pivot](https://go.microsoft.com/fwlink/?LinkId=271027).
  
## <a name="excel-services"></a>Excel Services

Excel Services consente di condividere cartelle di lavoro con altri utenti, interagire con i report, visualizzare ed esplorare informazioni in una finestra del browser e aggiornare i dati. Excel Services consente inoltre di condividere tutte o parti di cartelle di lavoro con altri utenti e di controllare quali informazioni vengono visualizzate in tali cartelle di lavoro. In SharePoint 2016, le funzionalità di Excel Services sono disponibili tramite Excel per il Web in Office Online Server. Excel Services richiede Office Online Server in ambiente locale. Ulteriori informazioni su [Excel Services](https://go.microsoft.com/fwlink/p/?LinkId=271028).
  
## <a name="excel-web-access-web-part-for-sharepoint"></a>Web part Excel Web Access per SharePoint

È possibile utilizzare la web part Excel Web Access per visualizzare contenuti Excel in un sito SharePoint.
  
## <a name="field-list-and-field-support"></a>Elenco dei campi e supporto per i campi

Excel per il Web consente di aprire l'elenco dei campi e i campi del pozzetto per i rapporti di grafico pivot e i rapporti PivotTable visualizzati nelle finestre del browser. Questa funzione facilita l'esecuzione di modifiche temporanee alle informazioni visualizzate nel rapporto di un grafico o di una tabella pivot senza dover aprire Excel. Per ulteriori informazioni, vedere [Business Intelligence in Excel Services](https://go.microsoft.com/fwlink/p/?LinkId=271029).
  
## <a name="filter-enhancements"></a>Miglioramenti del filtro

Sono state semplificate la creazione e la connessione di filtri dati a rapporti. È possibile regolare la dimensione e l'orientamento di un filtro dati. È inoltre possibile formattare i filtri dati applicando stili predefiniti oppure creandone di propri.
  
## <a name="filter-search"></a>Filtro di ricerca

È possibile eseguire una ricerca nei risultati dei filtri velocizzando così il passaggio a un dato elemento contenuto in un elenco lungo. Ulteriori informazioni sull'[utilizzo di filtri e filtri dei dati](https://go.microsoft.com/fwlink/p/?LinkId=271030).
  
## <a name="odc-file-support"></a>Supporto file ODC

I file ODC consentono di gestire, condividere e riutilizzare le connessioni dati alle origini dati dell'organizzazione. Le connessioni dati a determinate origini dati possono essere configurate e gestite centralmente dal personale IT e possono essere rese disponibili per qualsiasi cartella di lavoro che utilizza file ODC. È possibile mantenere insiemi di file ODC in posizioni centrali della farm di SharePoint ed è possibile modificare le cartelle di lavoro di Excel per l'utilizzo di questi file.
  
## <a name="performancepoint-services"></a>PerformancePoint Services

Non disponibile per i clienti SharePoint Online. PerformancePoint Services consente ai clienti di SharePoint Server 2013 di creare dashboard interattivi che consentono di visualizzare indicatori di prestazioni chiave (KPI) e virtualizzazione dei dati sotto forma di scorecard, report e filtri. Ulteriori informazioni su [PerformancePoint Services](https://go.microsoft.com/fwlink/p/?LinkId=271032).
  
## <a name="power-bi-web-part"></a>Web part Power BI

Power BI è una famiglia di strumenti di analisi business basata su cloud che consente a chiunque di connettersi, visualizzare e analizzare i dati con una maggiore velocità, efficienza e comprensione. Permette agli utenti di connettersi a una vasta gamma di dati dinamici attraverso dashboard intuitive, fornisce report interattivi e offre visualizzazioni molto efficaci. Per ulteriori informazioni su Power BI, vedere [Introduzione al servizio Power BI](https://go.microsoft.com/fwlink/?linkid=844671).
  
## <a name="power-pivot-for-excel-in-sharepoint"></a>Alimentazione Pivot per Excel in SharePoint

Con Power Pivot per Excel, è possibile eseguire analisi di dati avanzate e creare modelli di dati sofisticati. È anche possibile combinare insieme volumi notevoli di dati provenienti da varie origini, analizzare rapidamente i dati e condividere opinioni. Con SharePoint Online, è possibile utilizzare Excel per il Web per visualizzare e interagire con grafici, tabelle, rapporti di grafico pivot e rapporti di tabella pivot in una finestra del browser. È anche possibile utilizzare la web part Excel Web Access per visualizzare contenuti Excel in un sito SharePoint. Per poter visualizzare fogli Power Pivot in SharePoint, è necessario crearli in Excel. Per ulteriori informazioni sull'utilizzo di [Power Pivot in Excel](https://go.microsoft.com/fwlink/p/?LinkId=271036).
  
## <a name="power-view-for-excel-in-sharepoint"></a>Power View per Excel in SharePoint

 Con Power View per Excel, è possibile visualizzare e interagire con dati creati utilizzando visualizzazioni, animazioni e query avanzate altamente interattivi. Con SharePoint Online, è possibile utilizzare Excel per visualizzare e interagire con le cartelle di lavoro che contengono fogli di Power View in una finestra del browser. È anche possibile utilizzare la web part Excel Web Access per visualizzare contenuti Excel in un sito SharePoint. Per poter visualizzare fogli Power View in SharePoint, è necessario crearli in Excel. Ulteriori informazioni sull'utilizzo di [Power View in Excel](https://go.microsoft.com/fwlink/p/?LinkId=271035).
  
## <a name="quick-explore"></a>Esplorazione rapida

Esplora rapida consente alle organizzazioni di selezionare un valore in un rapporto di grafico pivot o in un rapporto di tabella pivot e di visualizzare ulteriori informazioni su tale valore. Se, ad esempio, si sta visualizzando un rapporto di vendite e si seleziona un valore che rappresenta la quantità totale delle vendite, è possibile utilizzare Esplorazione rapida per visualizzare le informazioni aggiuntive, come la quantità delle vendite nelle diverse categorie di prodotti o aree geografiche. Ulteriori informazioni su [Esplorazione rapida](https://go.microsoft.com/fwlink/p/?LinkId=271037).
  
## <a name="refresh-excel-workbooks-interactively"></a>Aggiornamento delle cartelle di lavoro Excel in modo interattivo

Consente di visualizzare i dati aggiornati in Excel per le cartelle di lavoro Web connesse a origini dati esterne, ad esempio i database MS SQL. Tenere presente che è necessario connettere le cartelle di lavoro a origini dati esterne in Excel prima di visualizzarle in Excel per il Web.
  
## <a name="sql-server-reporting-services-ssrs-integrated-mode"></a>Modalità integrata di SQL Server Reporting Services (SSRS)

Quando SQL Server 2012 Reporting Services è integrato con SharePoint, è possibile utilizzare le funzionalità avanzate di gestione centralizzata e collaborazione dei documenti per i contenuti di SQL Server 2012 Reporting Services. I report vengono archiviati nelle raccolte documenti di SharePoint, insieme ad altri report, ad esempio Excel per i file Web. Le autorizzazioni di SharePoint possono essere utilizzate per controllare l'accesso al contenuto di Reporting Services e gli utenti sono in grado di avviare Progettazione report, Progettazione modelli e Generatore report all'interno di una raccolta documenti. Per ulteriori informazioni, vedere [server di report di Reporting Services](https://go.microsoft.com/fwlink/p/?LinkId=271039).
  
## <a name="sql-server-reporting-services-ssrs-web-part"></a>Web part di SQL Server Reporting Services (SSRS)

La Web part SQL Server 2012 Reporting Services consente agli utenti di visualizzare un report di Reporting Services esistente nella propria web part in un sito di SharePoint. Se il report di Reporting Services contiene parametri, gli utenti possono connettere i filtri a tale report utilizzando le impostazioni della web part. Ulteriori informazioni su [Web part Visualizzatore report](https://go.microsoft.com/fwlink/p/?LinkId=271041).
  
## <a name="timeline-slicer"></a>Filtro dei dati della sequenza temporale

Un filtro dei dati della sequenza temporale è un controllo che si può aggiungere a un dashboard creato tramite Excel. Un controllo Timeline consente agli utenti di visualizzare le informazioni per un determinato periodo di tempo. Ulteriori informazioni su [Business intelligence in Excel Services](https://go.microsoft.com/fwlink/p/?LinkId=271042).
  
## <a name="use-and-interact-with-excel-workbooks-containing-data-models"></a>Uso e interazione con cartelle di lavoro Excel contenenti modelli di dati

A volte, le cartelle di lavoro di Excel contengono modelli di dati di Power Pivot che consentono di effettuare un'analisi dati potente, eseguire il mashup di grandi volumi di dati da varie origini, analizzare i dati rapidamente e condividere informazioni dettagliate.
  
## <a name="user-defined-functions"></a>Funzioni definite dall'utente

Le funzioni definite dall'utente (UDF) offrono la possibilità di utilizzare formule in Excel per le celle del foglio di lavoro Web per chiamare le funzioni personalizzate scritte in codice gestito. Le FDU sono una funzionalità di programmabilità avanzata che consente all'organizzazione di sviluppare e inserire codice personalizzato in Excel per i fogli di lavoro Web. È possibile creare FDU per chiamare funzioni matematiche personalizzate, ottenere dati da origini dati personalizzate nei fogli di lavoro e chiamare i servizi Web dalle FDU.
  
## <a name="visio-services"></a>Visio Services

Visio Services è un'applicazione di servizio che consente agli utenti di condividere e visualizzare i\*file di disegno di Microsoft Visio (. vsdx\*) e di disegno Web di Visio 2010 (con estensione vdw). Visio Services non è più disponibile con SharePoint Online. Visio Services continuerà a essere disponibile in SharePoint Server 2010, SharePoint Server 2013 e SharePoint Server 2016. Ulteriori informazioni su [Visio Services](https://go.microsoft.com/fwlink/p/?LinkID=271043).

Per visualizzare la disponibilità delle funzionalità tra i piani di Office 365, opzioni autonome e soluzioni locali, vedere [Descrizione del servizio SharePoint Online](sharepoint-online-service-description.md).
